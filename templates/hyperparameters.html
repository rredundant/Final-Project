Hyperparameters

Of the 12 classification models used, the gradient boosting classifier performed best. We built upon this by experimenting with hyperparameters:

Individually optimized hyperparameters 42.3%
No hyperparameters 46.7%
Subsequently optimized hyperparameters 47.5%

learning_rate

“The learning rate shrinks the contribution of each tree by learning_rate.” Effectively it de-emphasizes each tree.

N_estimators

“n_estimators represents the number of trees in the forest. Usually the higher the number of trees the better to learn the data. However, adding a lot of trees can slow down the training process considerably, therefore we do a parameter search to find the sweet spot.”

max_depth

“max_depth. This indicates how deep the built tree can be. The deeper the tree, the more splits it has and it captures more information about how the data.“

min_samples_split

“min_samples_split represents the minimum number of samples required to split an internal node. This can vary between considering at least one sample at each node to considering all of the samples at each node.“

min_samples_leaf

“min_samples_leaf is The minimum number of samples required to be at a leaf node. This similar to min_samples_splits, however, this describe the minimum number of samples of samples at the leafs.”

max_features

“max_features represents the number of features to consider when looking for the best split.”
